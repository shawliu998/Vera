# Vera 私测案件包规范

## 目的

该案件包用于验证真实律师工作流，不用于训练模型。材料必须已获得授权并完成
脱敏；不得包含无关第三人的身份证号、住址、联系方式、银行账号或医疗信息。

## 建议材料

- 起诉状、答辩状或现有争点说明；
- 合同及补充协议；
- 付款、发票、交付或验收记录；
- 与争议直接相关的邮件、聊天或函件；
- 法院送达、举证、开庭等程序材料；
- 已有证据目录或律师工作底稿。

支持 PDF、DOCX、XLSX、TXT 和 Markdown。每个文件上限 100 MB，每批最多
100 个文件。扫描 PDF 应保留原始页序，不要先由第三方云 OCR 改写。

## 入场预检

案件目录必须位于本机绝对路径，不能包含符号链接。文件权限应为仅所有者可
读写，例如 `chmod 600`；目录建议为 `chmod 700`。

```bash
cd "/Users/a1-6/Documents/new agent/backend"
VERA_PILOT_CASE_DIR="/absolute/path/to/anonymized-case" \
  npm run check:vera:pilot-case-pack
```

预检不会复制文件，也不会输出文件名、绝对路径或正文。报告使用相对路径哈希
定位文件，并检查扩展名、大小、PDF/Office 容器签名、ZIP 安全边界和权限。
任何 blocked 项都必须先处理，不能绕过后直接导入。

## 验收边界

真实案件验收至少记录：导入成功率、OCR 页准确性抽检、引用定位错误、遗漏的
关键事实、错误请求权或抗辩、期限人工更正、文书修改量、导出阻断原因、总用时
和每个律师接管点。一个案件通过不代表法律正确性或生产就绪。
