## 真实模型实际运行记录

公开事件仅展示工具名称与结果，不展示工具参数；因此下表只能依据 `tool_failed` 等事件判断失败，**不能证明调用没有无效参数**。所有已观察到的单次工具调用序列均不超过 10 次。

| 类别 | 实际输入与提示词摘要 | 已观察到的工具序列 / 结果 | 当前结论 |
|------|--------------------|--------------------------|----------|
| 合同审查（多轮 K2.7 Highspeed / K3） | 每轮仅输入 2 份文档，未进行第二次输入；提示词要求对所选合同材料进行审查并给出表格和/或草稿交付物。 | 曾见 `fetch_documents → run_contract_review`；另一次为 `list_workflows → fetch_documents → run_contract_review`；还有一次为 `read_document ×2 → list_workflows → run_contract_review → create_legal_memo → run_custom_extraction`。至少一次真实产生 Review 与 Draft（`tabular_review_created`、`draft_created`）。 | Job 最终 failed：完成后仍有多余工具调用和/或异步恢复错误，未进入 `complete`；真实 XLSX/DOCX 未做导出验证。 |
| 自定义提取 | 实际输入 2 份文档；自然语言提示词要求按自定义字段抽取并输出结构化结果。 | 自然语言提示词先调用 `run_custom_extraction`，随后出现 `assistant_tool_failed`。 | 未通过；严格 JSON 重试见下节，未能实际执行。 |
| 时间线 | 未进入任务调用阶段。 | 无时间线工具调用。 | 未运行；因后续 403 停止。 |

**结论：合同审查、自定义提取、时间线三类任务均未形成全绿真实模型冒烟；其中时间线尚未实际运行。**

## 实际 Prompt

合同任务的自然语言版本要求“先读取两份合同，运行内置批量合同审查；若仍在处理，继续检查同一个 Review，生成比较 XLSX 和风险 Memo DOCX”。在模型未按异步约定继续后，最后一次诊断改为显式序列：

```text
先调用 fetch_documents；然后只调用一次 run_contract_review（commercial_agreement）；
若 asynchronous=true，使用同一个 review_id 调用 get_contract_review，直至
review.status=complete 且 memo 不为 null；完成后立即结束。
```

自定义提取实际运行的自然语言 Prompt 为：

```text
请从我选择的材料中提取当事人、文件日期、金额、付款期限、解除权和管辖法院，
生成 XLSX。只需要表格，不需要备忘录。请自主完成，不要要求我第二次输入。
```

时间线 Prompt 已准备但因端点 403 未实际提交，不计为已运行：

```text
基于所选案件材料制作事件时间线 XLSX，并根据该时间线生成事实摘要 DOCX。
```

## 后续严格 JSON 提示词尝试

- 后续以严格 JSON 形式重试自定义提取时，Kimi 端点返回 `HTTP 403 access_terminated_error`，已停止重试，未形成新的任务调用结果。
- 因此，三类任务全部通过的真实模型冒烟尚未完成，仍是发布闭环项，不构成已完成验证。

## 环境限制

- **OAuth 令牌 15 分钟后过期**，长流程测试需要续期或缩短执行窗口。
- **Fake-IP 条件需要临时回环代理**才能稳定访问相关依赖。
