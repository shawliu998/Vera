# Vera 法律数据源供应商策略

日期：2026-07-15

## 1. 结论

Vera 不把“网站可访问”“存在试用账号”“有 API/MCP 文档”和“可以随客户端向客户提供内容”视为同一件事。供应商只有在技术接口、商业授权、数据使用范围和真实连接验证全部通过后，才能进入生产状态。

当前建议状态：

| 来源 | 官方公开接口 | 可技术 PoC | Vera 生产接入 | 默认禁区 |
|---|---|---|---|---|
| 北大法宝 | 已公开 MCP 和 CLI；Bearer Token | 可以，仅使用自有试用 Token、公开或脱敏查询 | 涉及嵌入式商业化、转授权、批量或数据授权时，必须取得企业书面授权 | 抓取会员网站、共享账号、未授权缓存/建库/转供 |
| 元典 | 已公开 REST API 和 Streamable HTTP MCP；API Key | 可以，仅使用自有 Key、公开或脱敏材料 | 必须补齐商业许可、数据使用附件、DPA 与 SLA 后才能随 Vera 分发 | 抓取网页、把试用积分或账号转给客户、未约定的缓存/转售 |
| 威科先行 | 中国版未发现公开第三方 API、SDK、数据 feed 或 MCP | 不做接口 PoC；只能产品评估、联系销售或人工深链 | 必须取得覆盖中国法律库的私有 API/数据 feed 与内容授权 | 爬虫、抓包调用内部接口、共享订阅账号、系统提取建库 |
| 官方来源 | 没有统一 API/MCP；必须逐站核验 | 可以人工检索、深链或导入合法取得的官方文件 | 仅可接入来源运营方明确提供并允许的 API/数据服务 | 把“公开网页”推定为“允许批量自动获取” |
| 人工导入 | 不需要供应商 API/MCP | 可以，本地导入和 OCR | 由用户对文件来源、使用权和披露范围负责；Vera 仍执行留存与导出门禁 | 自动标记为权威已验证、绕过版权/隐私/保密义务 |

“可技术 PoC”只证明接口能够工作，不证明 Vera 有权缓存全文、建立向量索引、把内容发送给模型、导出给客户或进行转授权。

## 2. 状态定义

### 可技术 PoC

同时满足以下条件时，才允许做技术 PoC：

- 接口、鉴权方式和测试额度由供应商官方公开；
- 使用 Vera 团队或测试机构自有的 Token/API Key，不共享个人账号；
- 查询只包含公开、合成或充分脱敏的测试材料；
- 不建立供应商语料库，不向最终用户开放，不宣传为正式数据源；
- 结果仅保留完成兼容性、字段和错误处理测试所需的最小时间。

### 需企业书面授权

出现下列任一情况，普通试用或个人/小团队订阅不足以支持上线：

- Vera 代多个用户、机构或租户调用；
- 在客户端、插件、Agent、MCP 或 OEM/白标产品中嵌入；
- 保存全文、摘要、分类、翻译、案例要旨或供应商生成的分析；
- 建立检索库、向量库、embedding、RAG 上下文或评测集；
- 将结果发送给本地以外的模型提供商；
- 导出 DOCX/PDF、分享链接、报告或批量数据；
- 向 Vera 客户再分发、转售或转授权；
- 需要生产 SLA、稳定版本、增量更新、删除通知或审计支持。

授权必须来自有权覆盖具体中国法律产品与具体内容字段的合同主体，并写入订单、数据使用附件或其他可归档书面文件。销售口头说明不能打开生产门禁。

### 不可抓取、不可假装已连接

Vera 不得使用浏览器自动化、Cookie、个人用户名/密码、抓包得到的内部接口、页面解析或其他未正式提供的方式获取供应商内容。受控网关不能把无授权抓取“洗白”为 API。

`configured` 只表示本地存在配置。只有真实的官方协议调用成功、身份和权限有效、来源主机匹配、数据使用政策已声明且生产激活门禁打开时，UI 才能显示“已连接”。没有安全健康检查时应显示 `configured_unverified`；接口、许可或门禁缺一时必须显示“不可用”或“需机构授权”。

## 3. 供应商核验

### 3.1 北大法宝

[法宝 MCP 官方平台](https://mcp.pkulaw.com/)和[接入指南](https://mcp.pkulaw.com/docs?doc=mcp-integration)公开了 MCP 与 CLI 两种接入方式。MCP 使用供应商提供的 Streamable HTTP 地址和 `Authorization: Bearer <token>`；CLI 使用同一 Token 获取结构化结果。官方页面将法规、案例检索和法条/案号识别等能力作为可调用服务。

[官方定价页](https://mcp.pkulaw.com/pricing)提供免费试用和自助次数包，同时明确企业版覆盖 API 与数据授权、私有化、SLA、定制接口，以及涉及转授权或嵌入式商业化的场景。因此：

- 技术 PoC：允许使用官方试用 Token 测试 MCP/CLI、字段映射、超时和错误处理；
- 生产：必须取得企业合同，逐项确认 Vera 客户端嵌入、BYOK/机构账号、多用户代调用、展示、缓存、RAG、导出和转授权；
- 禁止：使用北大法宝会员网页或未公开接口抓取内容。

[北大法宝服务协议](https://login.pkulaw.com/Register/ShowClause)把普通授权限定在合同和合理自用范围，未经书面许可禁止超权限向第三方提供、使用未对外提供的抓取方式，以及批量复制、镜像、下载或制作派生数据库。Vera 不得用 MCP 可调用这一事实覆盖这些使用边界。涉及真实案件时，还应按[隐私政策](https://login.pkulaw.com/Register/ShowPrivacy)和企业合同确认查询日志、请求内容与保留周期。

Vera 当前实现仅固定调用法规语义服务的 `search_article`：不动态加载工具，不声明案例覆盖，不把搜索摘要当作正文，`fetch` 在精确法条服务另行审计前始终 fail-closed。供应商平台拥有更多能力，不等于 Vera 已接入或获授权使用这些能力。

建议状态：`technical_poc_only`；取得并录入企业授权矩阵前，生产能力保持 fail-closed。

### 3.2 元典

[元典开放平台](https://open.chineselaw.com/)、[接口广场](https://open.chineselaw.com/api-square/)和[开发文档](https://open.chineselaw.com/docs/)公开列出法律法规、案例文书、企业信息和法律幻觉检测接口。REST 调用使用 `X-API-Key`；[MCP 接入页](https://open.chineselaw.com/mcp-config/)提供法律、案例和企业三类 Streamable HTTP Server，并使用 `Authorization: Bearer <API key>`。官方 [`llms-full.txt`](https://open.chineselaw.com/llms-full.txt)还说明 MCP 的 Accept header、业务错误码检查以及 Hermes 客户端关闭 sampling 的兼容要求。

[新站试用活动](https://open.chineselaw.com/activity/)可以支持自有账号的技术验证，但积分仅限本账户使用且不可转让，不能作为 Vera 向客户提供服务的授权依据。

[元典 SaaS 产品服务协议](https://www.chineselaw.com/ServiceContract.pdf)明确覆盖开放平台，保留平台内容权利，未经事先书面许可不得用机器人、爬虫、自动或手动程序复制、传播平台内容；同时不承诺搜索结果持续可用、正确或及时。公开接口解决了“是否有官方技术入口”，没有自动解决客户端缓存、RAG、展示、导出和转售范围，也没有提供生产 SLA。

[元典隐私政策](https://xszk.chineselaw.com/privacy.pdf)明确适用于开放平台，并说明会收集 API Key、API 调用日志和接口请求参数，用于鉴权、统计、安全审计以及维护、改进服务。因此真实案件查询上线前必须签署适用的 DPA/数据处理条款，确认输入留存、二次利用、分包商、删除、事件通知和数据地域；PoC 只能发送公开或脱敏内容。

Vera 当前已审计并路由的是 REST / `X-API-Key` adapter；元典 MCP 仅保留官方端点元数据，未进入客户端执行路径，启用前需独立的协议、工具面和回调安全审计。

建议状态：`technical_poc_only`；正式分发在商业许可、DPA、SLA 和内容权利矩阵完成前保持阻断。

### 3.3 威科先行

[威科先行® AI+·法律产品页](https://www.wolterskluwer.cn/solutions/Info/10029)确认其法规、案例、解读、智能检索、文档阅读、问答和合同工具能力，但截至本文件日期，威科中国官方产品页、WKinfo 页面和官方站点未公开第三方 API、SDK、数据 feed、开发者门户或 MCP 文档。

[WKinfo 服务条款](https://www.wkinfo.com.cn/login/termsOfService?format=&lang=zh_CN)把默认使用限定为内部管理、援引和参考以及限定数量的下载/保存；明确排除再发行、转发、出版、商业开发、系统提取建库，并禁止爬虫和其他自动下载程序、账号共享。[版权声明](https://www.wkinfo.com.cn/login/copyrightStatement?format=&lang=zh_CN)还说明内容权利可能属于威科中国或第三方许可方。

威科集团的[Legal & Regulatory U.S. 合作页](https://www.wolterskluwer.com/en/solutions/legal-regulatory/partnerships)确实提供 secure API、内容许可和技术集成合作，但页面明确属于美国法律与监管业务，不能据此推定覆盖威科先行中国法律库。

因此：

- 不做浏览器或内部接口 PoC；
- 通过产品页列出的 `CN-Support@wolterskluwer.cn`、`400-066-5518` 询问中国法律库 machine-to-machine API、增量数据 feed 或 OEM/内容许可；
- 只有取得覆盖具体语料、字段、用户、地域及 AI 使用方式的书面合同后，才实现正式 adapter；
- 若只获准实时查询和深链，则不得落全文或建立向量索引。

建议状态：`enterprise_license_required`，同时在未取得正式接口前保持 `unavailable`。

### 3.4 官方来源

优先核验的官方一手来源包括：

- [国家法律法规数据库](https://flk.npc.gov.cn/)；
- [国务院政策文件库](https://www.gov.cn/zhengce/zhengceku/bmwj/home.htm)；
- [国家行政法规库](https://xzfg.moj.gov.cn/)；
- [人民法院案例库](https://rmfyalk.court.gov.cn/)及最高人民法院的[上线说明](https://www.court.gov.cn/zixun/xiangqing/426222.html)；
- [中国裁判文书网](https://wenshu.court.gov.cn/)。

这些来源可以作为内容权威性和时效性核验入口，但“面向社会开放”不等于“提供第三方批量 API”。截至本文件日期，上述入口未形成一个可供 Vera 统一调用的官方 API/MCP 合同。默认策略是人工检索、深链、用户下载后导入；只有来源运营方明确发布接口文档和自动化使用条件，或与部署机构签订数据服务协议后，才可启用来源专属 adapter。

官方法规原文、网站整理结果、下载文件、案例文书和其中个人信息的权利与合规边界并不相同。Vera 必须保存发布机关、官方 URL、文号/案号、版本、公布/施行日期、检索时间和内容哈希，并对案例隐私、撤回、更新与删除执行额外门禁。

建议状态：`manual_or_link_out`；来源专属书面授权后可升级为 `licensed_api`。

### 3.5 人工导入

人工导入是 Vera 的本地来源能力，不是绕过供应商授权的手段。用户可以导入其自行起草、从官方来源合法下载、由客户提供或已获得使用许可的文件。

导入时必须：

- 记录原始文件、来源类型、发布机关/提供人、原始 URL、取得时间和内容哈希；
- 对扫描件使用本地 OCR，保留页码与原始版面锚点；
- 默认标记为 `captured_unverified`，经官方原文、案号和版本核验后才能提升验证状态；
- 按 Project 的保留政策控制本地全文、索引、模型使用和导出；
- 不因文件位于本地就推定可以对外传播或上传到远程模型。

人工导入不需要供应商 Token，但需要用户对来源权利、客户保密、个人信息和导出范围作出明确确认。

## 4. 独立授权门禁

每个来源、产品和合同分别记录以下字段。任何未声明项按禁止处理：

| 门禁 | 需要的证据 | 未确认时的默认行为 |
|---|---|---|
| API/MCP | 官方文档、固定主机、协议版本、字段与错误码 | 不调用 |
| MCP 工具面 | 固定 tool allowlist、Schema、sampling/callback 行为 | 不加载动态或未复核工具 |
| 鉴权 | 机构 service token/API Key、允许的用户与租户范围 | 不读取凭据；不共享账号 |
| 查询内容 | 供应商对请求日志、保留和二次利用的说明/DPA | 仅公开或脱敏测试；真实案件不外发 |
| 展示 | 可展示元数据、摘要、短引文或全文的明确范围 | 仅显示最小元数据与官方深链 |
| 缓存 | 允许字段、TTL、离线使用、终止/撤回删除规则 | 不保存正文 |
| RAG/embedding | 本地索引、远程模型、训练、评测和派生数据权利 | 不索引、不发送给模型 |
| 导出 | DOCX/PDF、短引文、全文、报告和分享范围 | 禁止导出供应商内容 |
| 转授权 | 多用户、最终客户、OEM、白标、转售和地域 | 不向 Vera 客户开放 |
| 更新与删除 | 增量、版本、废止、撤回、删除 feed | 不宣称时效完整；定期人工核验 |
| 生产保障 | SLA、限流、配额、变更通知、审计和支持 | 保持 PoC 或不可用状态 |

API 权利和内容权利必须分别确认。供应商允许调用搜索接口，不代表允许永久保存搜索结果；允许显示全文，不代表允许将全文用于 embedding、模型训练或向最终客户转供。

## 5. Vera 适配方式

供应商协议不同，Vera 不要求它们伪装成同一种外部 JSON API：

1. 对官方公开并获授权的协议，使用来源专属 adapter，例如 MCP、REST 或正式数据 feed。
2. adapter 只在 Vera 内部归一化为搜索、详情、溯源、版本和授权能力，不丢失供应商的原始 ID、URL、版本及许可元数据。
3. 对机构已有私有 API 或供应商定制接口，可使用[受控法律来源网关](authorized_legal_source_gateway.md)作为企业兼容模式。
4. 凭据仅保存于 Vera 的本地加密凭据路径，renderer 不回读明文；每次外发只发送律师确认的最小化查询。
5. provider 必须按合同声明 `display`、`retention`、`modelUse`、`export` 和 `onwardDistribution` 能力；不得从“接口返回了字段”推断使用权。
6. MCP adapter 必须固定服务主机、协议版本、工具名称和参数 Schema；默认拒绝供应商新增的动态工具、sampling、roots、prompt 或其他未经复核的 server-initiated 能力。
7. 任何 provider 失败都不得回退为浏览器抓取、模型编造引文或测试 fixture。

v2 运行时线上只投影 `basis` / `retention` / `modelUse` / `export`。`display` 与 `onwardDistribution` 仍属于上线前必须归档和人工复核的外部合同证据，尚未成为 v2 wire 字段。因此当前生产门禁必须保持关闭，不得把这两项描述为已有运行时强制。

## 6. 上线验收

正式启用一个供应商前必须完成：

1. 法务归档签署合同、订单、数据使用附件、DPA 和适用 SLA。
2. 产品负责人把合同权限映射到上述独立门禁，并由第二人复核。
3. 安全负责人核验固定主机、TLS、凭据存储、日志脱敏、超时、限流和撤销。
4. 使用不含客户信息的受控查询完成真实连接、字段、版本和删除测试。
5. 验证缓存到期、来源撤回、Project 删除、备份恢复、模型调用和 DOCX/PDF 导出都执行相同授权门禁。
6. UI 只显示经真实测试的状态，并清楚标注试用、机构授权、连接未验证和不可用。

任一步骤缺失时，Vera 保持 fail-closed；供应商名称可以作为规划项显示，但不得描述为“已接入”或“已连接”。
